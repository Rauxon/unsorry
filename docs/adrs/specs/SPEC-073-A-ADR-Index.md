# SPEC-073-A: ADR Index Generator and Post-Merge Refresh

Implements: [ADR-073](../ADR-073-ADR-Index-Generated-README.md) · Status: Living · Updated: 2026-06-18

## What this adds

A generator that turns the `docs/adrs/ADR-*.md` corpus into an index, plus a workflow
that keeps the index in sync on `main`.

- **`tools/adr_index/`** (`generate.py` + `__main__.py`), tests in `tools/adr_index/tests/`.
- **`.github/workflows/adr-index.yml`** — post-merge refresh.
- **Generated artifacts** (committed, banner-marked "do not edit by hand"):
  - `docs/adrs/README.md` — Markdown table.
  - `docs/adrs/adrs.json` — structured index.

## Parsing (per `docs/adrs/ADR-*.md`)

The source is a **non-recursive** glob of `docs/adrs/ADR-*.md`, which naturally excludes
`README.md`, `adrs.json`, and the `specs/` subdirectory.

| Field | Source | Fallback |
|---|---|---|
| `number` | leading `ADR-(\d+)` of the filename | file skipped if absent |
| `id` | `ADR-{number:03d}` (zero-padded) | — |
| `title` | first `# ` H1, with a leading `ADR-NNN:` prefix stripped | filename: drop `ADR-NNN-` + `.md`, hyphens → spaces |
| `status` | `\| **Status** \| <value> \|` row, kept verbatim (incl. any parenthetical) | `Unknown` |
| `date` | `\| **Date** \| <value> \|` row | `—` |
| `file` | the filename within `docs/adrs/` | — |

Rows are sorted by `(number, file)`. **Duplicate ADR numbers** (the corpus has two
`ADR-041`s) are all retained as separate rows; the generator prints a non-fatal
`::warning::duplicate ADR number NNN: <files>` to stderr and exits 0.

## Output contract

- **`README.md`**: a `<!-- GENERATED by ... -->` banner, an `# Architecture Decision
  Records` heading, an intro linking ADR-001 and `../protocols.md`, then a table with
  header `| ADR | Title | Status | Date |`. The ADR cell links to the file
  (`[ADR-001](ADR-001-...md)`). Cell `|` characters are escaped (`\|`).
- **`adrs.json`**: `{ "generated_by": <str>, "count": <int>, "adrs": [ {id, number,
  title, status, date, file}, ... ] }`, 2-space indented, `ensure_ascii=False`.
- **Determinism (load-bearing):** output embeds **no wall-clock timestamp**, so
  `--check` can compare rendered content byte-for-byte against the committed files —
  the same drift model as the leaderboard / targets / queue generators. Both files end
  with a trailing newline.

## CLI

```
python3 -m tools.adr_index [<root>]           # README Markdown to stdout
python3 -m tools.adr_index --json [<root>]    # adrs.json to stdout
python3 -m tools.adr_index --write [<root>]   # write docs/adrs/README.md + adrs.json
python3 -m tools.adr_index --check [<root>]   # exit 1 if either artifact is stale, else 0
```

`<root>` defaults to `.`. The mode flags are mutually exclusive (two → exit 2).

## Workflow (`adr-index.yml`)

On push to `main` touching `docs/adrs/**`, `tools/adr_index/**`, or the workflow itself
(plus `workflow_dispatch`):

1. `python3 -m tools.adr_index --check .` → if in sync, stop.
2. on drift **and** `REFRESH_TOKEN` present: a retry-rebase loop (5 attempts) that
   resets to latest `origin/main`, `--write`s, and commits
   `docs/adrs/README.md docs/adrs/adrs.json` as `github-actions[bot]` with
   `docs: refresh ADR index [skip ci]`, then pushes to `main`.
3. on drift but no `REFRESH_TOKEN`: a `::warning::` report-only step (issue #417).

`concurrency: { group: adr-index-refresh, cancel-in-progress: false }`;
`permissions: contents: write`; pinned action SHAs (ADR-019). The refreshed files are
*within* the trigger path, so the `[skip ci]` commit message is what stops the refresh
commit from re-firing the workflow.

## Acceptance criteria

1. `python3 -m tools.adr_index --write .` writes both artifacts; `--check .` then exits 0.
2. Mutating any ADR's `**Status**` makes `--check .` exit 1 and name the stale file(s).
3. Both `ADR-041` files appear as distinct rows; a duplicate-number warning is emitted.
4. An ADR with a missing Status/Date renders `Unknown` / `—` rather than crashing.
5. `adr-index.yml` is valid, pinned, `permissions: contents: write`, triggers on the ADR
   paths, and on drift commits both artifacts with `[skip ci]`.
6. `python3 -m pytest tools/adr_index -q` is green; Gate B (`tools.gate_b validate .`) passes.

## Operational note

Same requirement as `targets-board.yml` / `queue-board.yml` (#417): the push to `main`
needs the `REFRESH_TOKEN` admin secret. Without it the index simply isn't auto-refreshed
(the workflow warns instead of failing); a maintainer can always run
`python3 -m tools.adr_index --write .` locally and commit.
