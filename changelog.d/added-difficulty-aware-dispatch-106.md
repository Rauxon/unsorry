**Difficulty-aware dispatch (ADR-106 / SPEC-106-A).** The queue dispatcher now deprioritises low-difficulty *template* proofs: high-difficulty branches dispatch (with ADR-075 per-solver round-robin) *before* template floods (`template-*`, `ring`, `zmod-decide`, …), so the genuinely hard work that the leaderboard difficulty-weights — the v2.0.0 benchmark suites — reaches Gate A ahead of a single solver's week-old template backlog. A branch with no/unknown model is treated as high (fail-safe). Reorder-only — dedup (ADR-064/071), the governor (ADR-058) and Gate A still decide what merges. Logic extracted to the unit-tested `tools/dispatch/fair_order.py`; reversible via `UNSORRY_DIFFICULTY_DISPATCH=0` (fairness-only) / `UNSORRY_FAIR_DISPATCH=0` (lexical). Refs #5678.
