**Branch hygiene — prune merged/closed-PR branches + fix the superseded janitor.** `delete_branch_on_merge` is off, so every squash-merged PR left its head branch behind (squash makes the tip a non-ancestor of `main`, so a `--merged` prune can't see it) — ~1.1k `feature/*` accumulated. The `stale-branch-janitor` now also runs `tools.repo.merged_pr_branches`, deleting branches whose PR is merged/closed (protected refs — `main`/`claims`/`gh-pages`/`queued/*` — are never touched; `queued/prove/*` stays owned by the goal-status prune). Also flips `superseded-pr-janitor` to `cancel-in-progress: false` so the merge firehose stops cancelling it before it can close superseded PRs (it was completing ~10% of runs). Refs #5678.
