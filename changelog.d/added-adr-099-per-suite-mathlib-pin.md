Proposed **ADR-099 + SPEC-099-A (Per-Suite Mathlib Pin for Benchmark Suite Ingestion & Verification)**: a registered benchmark suite's `(toolchain, mathlib rev)` becomes authoritative, so ingestion and verification evaluate each suite **at the mathlib version it was written for** instead of the single repo-wide pin. The importer (`import_benchmark --build`) elaborates/builds each suite in a suite-scoped lake project (`targets/<suite>/_verify`, reusing the archive-package scaffolding) at its native pin, dropping pin-drift quarantines (e.g. `putnam-1966-b5`'s removed `Finset.toSet`, the IMOLean 2022–25 bulk) toward ~0; benchmark proofs verify and record in a segregated track kernel-checked **at the suite pin** (sound under ADR-048/049 — the kernel is the sole oracle at any mathlib version, so "0 false positives (kernel-verified)" still holds), surfaced per suite under the existing `cohort:benchmark`; and `./swarm/run.sh --goal <slug>` selects the suite's toolchain. Rejects porting statements to the repo pin (today's `--build` behaviour — lossy and unfaithful). Accepts N mathlib binary caches (one per pin), mitigated by the FRO `lake exe cache get`. Status: Proposed. (#6381, #5643 M8.)
