`./swarm/run.sh --goal <slug>` now **proves a benchmark goal in its suite's verifier context — at the suite's own mathlib pin — instead of the repo-wide pin** (ADR-099 / SPEC-099-A §3, #6381). A new `tools/intake/suite_context.py` resolves a goal id to the registered suite that owns it and that suite's `(toolchain, mathlib rev, _verify project, lake target)`; the swarm prover (`swarm/agent.sh`) consults it so that, for a benchmark goal, the cache warm-up, the proof module, the ADR-011 binding obligation, the `--wfail` local verification, and the proof-PR all run in/target the suite's `targets/<suite>/_verify` lake project (the repo `axiom_audit` is skipped there — Gate A's `gate-a-benchmark` leg verifies at the suite pin instead). Organic goals are byte-identical to before (empty suite context → repo pin). No manual toolchain setup is needed: elan auto-installs the suite's (often older, e.g. v4.24) Lean toolchain per-directory from `_verify/lean-toolchain` on first build, alongside the repo's v4.30 (ADR-002; elan itself is bootstrapped by `ensure_lake`, ADR-097) — only the pin's mathlib oleans are fetched separately via `lake exe cache get`. Covered by a new `agent.sh --self-test` case and `tools/intake/tests/test_suite_context.py`.
