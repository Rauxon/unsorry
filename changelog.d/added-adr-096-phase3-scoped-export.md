Proposed **ADR-096 + SPEC-096-A (Phase 3 — declaration-scoped export + independent checker as a kernel-diverse anchor)**: records the decision following the ADR-093 / D2 pilot (#5684). The pilot showed the ~5.9 GB whole-module export was an artifact of `import Mathlib`; a **declaration-scoped** `lean4export` (the proof's theorem + its transitive *declaration* closure) is **2.7–46.5 MB** (126×–2194× smaller), and an independent Lean kernel (`nanoda`) checks it in **0.08–2.6 s** (4×–160× faster than `leanchecker`), deterministically, **accepting valid proofs** (target-confirmed, 20/20) and **rejecting ill-typed ones** (negative control 5/5 — a proof-value swap that breaks the statement/term match). ADR-096 adopts this as Phase 3's validated mechanism, introduced conservatively as a **non-gating, kernel-diverse defense-in-depth anchor** strictly subordinate to ADR-049 — the Lean kernel stays the sole `p = 1` truth oracle; the independent checker is additive, never a replacement. Lowering central `p < 1` is explicitly out of scope (a separate future ADR). Carries acceptance gates (broader red-team, checker code review, cross-machine determinism, hardest-proof stress) before enablement. Status: Proposed.
