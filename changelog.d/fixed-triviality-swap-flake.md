Fixed the `triviality` check intermittently failing (and blocking merges) on its "Add swap headroom" step with `fallocate: Text file busy`. On a reused GitHub runner `/mnt/swapfile` can already be active swap, so re-creating it failed and reddened the whole (otherwise-advisory) triviality job. The step is now idempotent and best-effort: it skips when swap is already active and never fails the job over swap setup (swap is only OOM headroom for the changed-goal probe). Removes a recurring infra flake that was stalling PRs — including auto-archive cut PRs whose merge shrinks the active set.
