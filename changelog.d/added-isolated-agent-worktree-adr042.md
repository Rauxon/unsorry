The swarm agent (`swarm/agent.sh`) now **runs in a dedicated per-agent worktree** by default (ADR-042/SPEC-042-A). Previously the agent synced, built, and claimed in the directory it was launched from — so an uncommitted edit there broke `sync_repo`'s `--ff-only` merge and killed the cycle, and two agents in one checkout raced. At startup (before any provider/auth/model resolution) the agent relocates into `$UNSORRY_WORKDIR/agent-main-<agent-id>` — a detached checkout of `origin/main` created with `git worktree add`, reused across cycles so its `.lake`/mathlib build cache is paid once — and `exec`s the worktree's `agent.sh`. Inside the worktree `sync_repo` advances by `git reset --hard origin/main` (re-entrant) and `require_main_checkout` accepts the detached HEAD (`require_main_matches_origin` still enforces HEAD == origin/main). The operator can now edit proofs and the harness, and run `--prove-local`, in the launch dir while agents run, and run several agents per host without interference. Opt out with `UNSORRY_NO_ISOLATE=1` (runs in place; launch dir must be on main); override the path with `UNSORRY_AGENT_WORKTREE`. `--self-test` and `--prove-local` never relocate. Self-tests `test_relocate_into_worktree_noop`, `test_require_main_checkout_isolated`, `test_ensure_agent_worktree`.
