Adopted **verify-on-ingest** for archive validation (ADR-048): an archive-block PR now validates **provenance + packaging** — byte-identity to the already-verified active proof (ADR-018, in `gate-a-prepare`) plus `lake build --wfail` packaging sanity — and **no longer re-runs `leanchecker` kernel replay** on the archived proofs. Those proofs were kernel-verified by Gate A when they were active and are byte-identical now, so re-replaying the same immutable artifact re-proves nothing; it also loaded each archive package's own full mathlib image and OOM-killed even a 16 GB runner (#764). This removes the archive-OOM class entirely (archive PRs fit the 8 GB runner) and supersedes the tactical chunking (#823) and 16 GB-pin (#838) attempts. Lean soundness is unchanged — each proof is still kernel-replayed once, at the PR that introduces it, under its toolchain/mathlib context, and a context change (toolchain/mathlib/lake/checker) still forces re-verification; a scheduled full replay remains as defense-in-depth. The residual risk shifts to bookkeeping (the system must know the archived file is exactly the verified one), guarded by ADR-018 byte-identity. Phase 2 (push-to-main full replay → scheduled/incremental + downscale CI 16 GB → 8 GB) is a documented follow-up.
