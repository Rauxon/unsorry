Lowered the auto-archive ceiling from 40 to 20: the active proved set is now kept at ≤20 proofs instead of ≤40. The dominant Gate A memory peak (`lake build UnsorryLibrary --wfail`, the axiom audit, and the leanchecker replay closure) scales with the active-library size; halving it keeps each run well inside a 16 GB runner and removes the OOM/runner-death class that was orphaning runs as zombies. Archiving simply triggers about twice as often (~every 20 proofs); archived proofs are validated by provenance + packaging, not re-replayed (ADR-048), so the extra cuts are cheap, and the one-archive-PR-at-a-time guard still prevents flooding Gate A.
