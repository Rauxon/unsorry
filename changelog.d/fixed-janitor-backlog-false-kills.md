Fixed the stale-run janitor cancelling healthy Gate A runs under a deep backlog (which tripped the fail-closed replay/cover asserts and left ~22 PRs showing `gate-a` FAILURE). The janitor measures run *age*, so a run that sits `in_progress` while its replay shards queue >60 min for one of the few lanes — legitimate, not a zombie — was being cancelled. Raised the `in_progress` threshold 60 → 180 min (a true zombie wedges `in_progress` forever, so the higher bar still reaps it) and made it tunable without a code change via the `UNSORRY_JANITOR_INPROGRESS_MIN` repo variable. This pairs with the event-driven trigger (which made the janitor fire far more often, amplifying the false kills).
