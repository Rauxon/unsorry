Hardened the auto-archiver so one inconsistent goal can no longer poison a whole block (and stall all archiving). A recompose/shared proof can place a goal's statement-binding theorem in a module owned by another goal; if that module is left active or was archived in an earlier block, the cut package fails Gate A's binding validation in isolation ("no library module declares '<name>'" — e.g. `nat-sq-lt-two-pow-s2`, which broke block 0033). `select_block` now runs a CI-faithful pre-check (`_binding_unsatisfiable_in_block`, mirroring `check_statement_binding`'s non-private `theorem/lemma` resolution) and **defers the whole decomposition component** of any goal whose binding theorem would not be declared inside the package, iterating until the block is self-consistent. The rest of the block still cuts, so archiving keeps running.
