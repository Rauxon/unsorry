Added the **broader red-team mutation suite** to the export-checker harness (ADR-096 / SPEC-096-A §4.1, acceptance gate 1): beyond the existing value-swap negative control, nanoda is now exercised against **type-swap** (altered statement), **dangling-ref** (structurally-corrupt export), and **axiom-restrict** (empty permitted-axiom whitelist → any axiom use rejected; the enforcement path that also stops a sneaked `sorryAx`). Exposed via `tools.pilot.export_checker_pilot --red-team` and a `red_team` input on the export-checker-pilot workflow, with hermetic unit tests and a per-class report. The spec records the **honest boundary**: genuine *vacuity* (a well-typed proof of a weaker statement passed off as the goal) is correctly accepted by any kernel checker — catching it is the ADR-011 `*Binding` gate's job, so that gate remains a non-negotiable companion to any nanoda-as-gate placement.
