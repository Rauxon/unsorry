**Fixed: the full axiom audit crashed on goals sharing a top-level name.** Two benchmark goals (`brualdi-ch1-10`, `brualdi-ch1-16`, from #6371) each declare a top-level `structure IsMagicSquare`, so `axiom_audit` importing them into one environment failed (`environment already contains 'IsMagicSquare.mk'`) — breaking every full-audit run (the daily backstop and any gate-tooling PR, e.g. the nanoda-cache PR). ADR-018 freezes goal `.lean` files, so the *audit* must tolerate the clash: the goal-audit driver now batches goals **collision-free** (`collision_free_chunks`) — clashing goals go to separate `axiom_audit` calls — instead of `split_evenly`, which could co-locate them. Soundness-neutral: a goal's axioms don't depend on what else shares its audit call.
