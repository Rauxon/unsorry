The advisory kernel-diverse independent check (ADR-096) is now **on by default** in the client entry scripts (`swarm/run.sh`, and a standalone `swarm/supervise.sh`): every run re-checks each proof with an independent Lean kernel (nanoda) over a declaration-scoped lean4export, bootstrapping the tools on first use (Rust auto-installs if missing). Opt out with `--no-independent-check` (alias `--no-nanoda`) or `UNSORRY_INDEPENDENT_CHECK` set to a falsey value (infra/CI can disable without changing args). `--independent-check` is still accepted (now redundant). It remains strictly **non-gating** — admits nothing, never blocks proving (a failed bootstrap or disagreement is logged, never fatal); ADR-049's p=1 Lean gate in CI is unchanged. run.sh sets `UNSORRY_INDEPENDENT_CHECK` to 0/1 explicitly so the decision propagates to supervise.sh/agent.sh.
