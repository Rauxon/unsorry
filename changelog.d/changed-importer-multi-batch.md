`import_benchmark` is now idempotent and multi-batch: it **skips statements already imported** (so a re-run picks up the next `≤--limit` obligations) and **accumulates the suite skeleton across batches** into one ADR-078 target, instead of each batch replacing it. This lets a suite larger than the 50-per-PR cap (e.g. all 672 PutnamBench problems) be ingested across successive PRs into a single growing target. (#5643.)
