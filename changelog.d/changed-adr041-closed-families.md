Amended ADR-041 (Proof Archive Blocks) with the **"archive closed proof families only"** policy: a decomposition family (parent + sublemmas + proofs + decomposition metadata + telemetry) is archived as one atomic unit and only when every member is proved; a family with any open member stays active. Rationale recorded: (1) Gate B atomicity — a decomposition + parent + subs must validate within one tree (GB008/GB016); (2) recomposition inputs must stay active — the active recompose/unblock/`proved-deps` machinery and Lake build see only active `library/`, so archiving an open parent's proved sublemmas would hide its recomposition ingredients (a non-issue in practice, since an open-parent-all-subs-proved family is a transient recompose-candidate that closes before it's archivable). Also documented, as **deferred** to its own ADR/PR, archive-aware recomposition / dependency reuse (the 6 cross-cutting changes to make archived lemmas importable by new active proofs) and the related `retired`-parent-status gap (so genuinely-stuck families don't pin the active replay surface). No behaviour change; the cut already enforces family-atomicity (SPEC-041-A §8).
