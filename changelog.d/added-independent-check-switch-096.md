Added an opt-in **contributor-side independent (kernel-diverse) proof check** (`UNSORRY_INDEPENDENT_CHECK`, ADR-096 Phase 3a). When enabled, after a proof verifies locally (`prove_local_verify`), `swarm/agent.sh` re-checks it with an independent Lean kernel (`nanoda`) over a declaration-scoped `lean4export` of the proved theorem — a second-kernel confirmation built on the validated D2 pilot (#5684). It is **advisory and NON-GATING**: it admits nothing, never blocks proving (a missing toolchain or a disagreement is logged/warned, never fatal), and is strictly subordinate to ADR-049's `p = 1` Lean gate in CI, which is unchanged. Default **off**; build the tools once with `tools/independent_check/setup.sh` (sets `LEAN4EXPORT_BIN`/`NANODA_BIN`). New `tools/independent_check/` reuses the pilot primitives (DRY); covered by Python tests + an `agent.sh --self-test` case. The throughput win comes later from a trusted-side placement; this entry generates agreement data and exercises the path at swarm scale with zero risk.
