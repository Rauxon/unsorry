Fixed the queue dispatcher (`agent.sh --dispatch-queue`) stalling for ~9 minutes per pass and opening **zero** PRs on a large queue. The ADR-064 goal dedup checked open-PR membership with a per-branch `gh ... --search` call, but the GitHub search API allows only **30 requests/min** — across a ~283-branch queue at `UNSORRY_DISPATCH_LIMIT` ≥ 10 it exhausted that bucket and stalled the whole pass on retry backoff (`9 min ≈ 283 ÷ 30`). The check is now resolved from a **single** `gh pr list --state open` call (`dispatch_open_pr_goals`, core quota 5000/h) made once per pass and tested by in-memory string membership — O(1) API cost regardless of queue size. Dedup behaviour is unchanged (proved-on-main, open-PR, and same-pass goals are still skipped); a full real dispatch pass now completes in seconds instead of timing out.
