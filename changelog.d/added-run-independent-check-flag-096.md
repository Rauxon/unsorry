Added a `./swarm/run.sh --independent-check` flag (ADR-096 Phase 3a) — the one-step way to enable the advisory kernel-diverse check. It **bootstraps the tools on first use** (builds the pinned `lean4export` + `nanoda` via `tools/independent_check/setup.sh`, needs lake + cargo), enables the per-proof `nanoda` re-check, and strips itself from the args passed to the prover. Omit it for a normal run with zero overhead; setup failure (no lake/cargo) downgrades to a warning and proves without it. NON-GATING — admits nothing, never blocks proving; ADR-049's `p = 1` Lean gate in CI is unchanged. (The underlying `UNSORRY_INDEPENDENT_CHECK` env var still works for advanced users managing the binaries themselves.)
