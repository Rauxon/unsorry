**Gate A — cache the pinned independent-check binaries.** The `gate-a-nanoda` job rebuilt lean4export + nanoda from source on every run (~3 min), even though both are pinned to fixed commits and the kernel check itself takes ~9 s. It now caches `~/.unsorry/independent-check` keyed on the two pin sources (`lean-toolchain` + `tools/independent_check/setup.sh`), so a cache hit makes the build a no-op and the job drops to roughly the check time. A pin bump changes the key and forces a fresh build, so no stale binary can be served; soundness is unchanged (leanchecker replay still gates every proof at p=1, ADR-048's daily full audit backstop still runs). Refs #5751, #5678.
