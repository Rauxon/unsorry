Added `tools/gate_a/nanoda_gate.py` (ADR-097 / SPEC-097-A): the Gate A axiom-audit replacement driver. It reads `compute_audit_targets` (the single source of truth for what to audit) and, for the changed library closure, runs a declaration-scoped `lean4export` + nanoda check with the axiom whitelist hard-enforced (`unpermitted_axiom_hard_error`, so a sneaked `sorryAx` fails) plus the `pp_declars` positive control — the same footprint check the `axiom_audit` performs, in ~seconds on one runner. A non-leaf / global-impact / untrusted diff reports `covered=False` so the caller falls back to the real audit (fail-closed). Driver + tests only — not yet wired into the gate. `check_proof` gained an `enforce_axioms` option.
