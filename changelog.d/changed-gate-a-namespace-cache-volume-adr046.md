Gate A now caches the whole `.lake` tree on a **Namespace cache volume** (keyless bind-mount via `namespacelabs/nscloud-cache-action`), superseding the `actions/cache` build cache from ADR-045 (ADR-046/SPEC-046-A). The `actions/cache` approach never warmed on `main` (its runs are skipped or cancelled), made the three parallel jobs collide on cache reservation (*"Unable to reserve cache… another job may be creating this cache"*), and never cached mathlib — so every run re-downloaded ~2.4 GB of mathlib oleans from Azure (~19 min) and cold-built the library. The volume persists mathlib oleans, resolved dependency clones, and the `UnsorryLibrary`/`UnsorryGoals` oleans across runs on fast local storage: no re-download, no re-clone, incremental build — with no key races and no PR/main scoping. lean-action's `use-github-cache` is turned off (the volume replaces it). The cache step is `continue-on-error`, so gate-a **soft-fails to a cold run** if no cache volume is attached. Soundness is unchanged: Lake's content-hash traces recompile any changed source, and the axiom audit + `leanchecker` kernel replay still validate every olean that reaches `main` (full replay on push to `main`), whether built or volume-restored. **Operator step:** enable caching (≥20 GB) on the `namespace-profile-unsorry-1`/`-2` runner profiles; until then gate-a runs cold (covered by the #599 45-min timeout).
