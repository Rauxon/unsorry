Added an operator preflight doctor (`python3 -m tools.repo.doctor`, [ADR-065](docs/adrs/ADR-065-Operator-Preflight-Doctor.md) / [SPEC-065-A](docs/adrs/specs/SPEC-065-A-Operator-Preflight-Doctor.md)) — a first slice of the ADR-051/ADR-056 operator surface. It runs cheap, side-effect-free health checks and exits non-zero on any failure. The flagship `pr-token` check verifies the active token can actually open pull requests by POSTing to the pulls endpoint with no `head`/`base` (rejected at validation as HTTP 422 → never creates a PR, but only after authorization, so an unauthorized token returns HTTP 403 first). The scheduled `queue-dispatcher` workflow now runs it as a fail-fast preflight, so a `REFRESH_TOKEN` lacking pull-request-write fails the job in one step with a fix-it message instead of silently dispatching zero PRs and dying on `Resource not accessible by personal access token (createPullRequest)`. The check registry is extensible for future operator checks (runner capacity, last dispatcher run).
