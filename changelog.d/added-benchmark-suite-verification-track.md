Added a **segregated benchmark verification track** that kernel-verifies a suite's discharged obligations **at the suite's own mathlib pin** (ADR-099 / SPEC-099-A §2, #6381). Benchmark proofs cannot land in the repo `UnsorryLibrary` (one pin), so they live in a suite-scoped verification package (`targets/<suite>/_verify`) built at the suite's `(toolchain, mathlib rev)`. A new `tools/gate_a/benchmark_packages.py` validator (reusing the archive-package build helpers) runs `lake exe cache get` + `lake build --wfail` + a forbidden-token scan in each suite package — unlike archive packages, these proofs were never active in the repo, so each is kernel-built here rather than trusted via byte-identity. A new `gate-a-benchmark` Gate A leg invokes it on changed suites (N mathlib caches, one `cache get` per distinct suite pin, via the FRO cache). `"0 false positives (kernel-verified)"` holds at the suite pin (ADR-048/049 — the kernel is the sole oracle at any mathlib version). `registered-targets.json` now counts obligations proved at the suite pin (from `targets/*/_verify/library/index`), and the `cohort:benchmark` segregation keeps them off the organic leaderboard.
