Fixed the unblock sweep (ADR-009) opening duplicate `unblock(<parent>)` PRs when several agents run concurrently. Each agent has its own session-local `SWEPT` set, so N agents seeing the same unblockable parent each opened a re-open PR (observed: 10 identical `unblock(six-dvd-pow-three-add-five-mul)` PRs, #1357 + 9 dupes). `unblock_sweep` now applies the same open-PR guard the prove path already uses (ADR-017): it skips a parent that already has an open `unblock(<parent>)` PR from any agent, so the agents open one PR between them instead of N. Factored the prove guard into a generic `open_kind_pr_exists` with `open_prove_pr_exists`/`open_unblock_pr_exists` wrappers; both keep the exact title-prefix match that avoids the #198 sibling-token false positive.
