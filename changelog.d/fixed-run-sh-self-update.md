`swarm/run.sh` now pulls latest and re-execs itself before guarding credit or starting any loop, so a long-lived launcher can no longer keep running stale code. Previously `run.sh` never fetched — only the loops it spawns (`agent.sh`, `supervise.sh`, `sourcing.sh`) self-synced — so a launcher started before the credit guard existed (or any other `run.sh` change) kept running guard-less indefinitely, since its own code is fixed at launch time. It now fetches `origin/main`, fast-forwards a clean checkout, and re-execs when its own blob changed (the same #428 harness-staleness self-re-exec `agent.sh` already does for itself). Best-effort and fail-open: an offline fetch or a dirty/diverged tree is left untouched so the swarm never fails to start, and `UNSORRY_RUN_NO_SELF_UPDATE=1` opts out. New pure `run_harness_stale` helper, unit-tested in `run.sh --self-test`.
